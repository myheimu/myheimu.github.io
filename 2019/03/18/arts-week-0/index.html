Review: 《My journey from Python to Go》

https://medium.com/appsflyer/my-journey-from-python-to-go-3859783c6b3c

作者先前主要写Python代码，现在转向Go，遇到了一些问题和一些挑战。

  1. 项目结构
    python是无所谓项目结构的，可以通过import做各种拐弯抹角的引用。而Go则有标准的结构，刚使用有些复杂，但对于较大型的项目则显得很有管理。

  2. 静态类型与强类型
    Python是典型的动态类型语言，而Go作为静态类型语言，会让Python用户感到约束。但用久了之后，动态语言带来的各种不确定性和处理复杂性，的确输给静态类型。

  3. 原生并行化
    Go语言通过goroutines和channels提供了原生的并行化支持，比如如下的一个例子

  1. 更好的JSON支持
    Go语言对JSON到GO Struct之间的转换提供了很多方便。

  2. 自带代码清理
    Go语言编译器能够自动清理不需要的变量等等。

  3. 寻找正确的框架
    Go语言和周边开发者提供了很好、又不需要各种比较选择的框架，相比起Python里面的多种不知如何选择，更加方便，例如：

  • Python的Requests，在Go里面选择net/http作为http请求器
  • Python的Flask、Jinja2，在Go里面选择Gin作为HTTP web框架
  • Python里面的各类CLI Creation，在Go里面选择Cobra作为命令行参数处理器。

最后，作者总结说：

作为五年的Python狂热用户，他对转换到Go语言确实感觉到了一些痛苦。

但他对Go的强大社区、贡献社和维护者感到非常兴奋，也非常愿意相信Google对Go的巨大推动力！

Tip: TensorFlow-CPU各预编译包与自编译优化（MKL、cpu优化）实际性能测试

最近在入门Tensorflow，针对tensorflow预编译包和自行编译有一些体会。

  1. 抛开tensorflow-gpu包不提，针对cpu部分。
    pip上的tensorflow包，是没有进行mkl优化的。
    conda上的tensorflow包，是进行了mkl优化的（从1.9版本开始）。
    如果想用pip安装mkl优化版本，则可以pip install intel-tensorflow。

  2. intel对自己的mkl很有信心，强烈推荐，还认为AVX2等指令集对tf性能影响不大，mkl才是最终大杀器。Intel® Optimization for TensorFlow* Installation Guide

  3. 网上有不少文章吹捧mkl，还进行了测试，认为mkl版本能提高1倍到4倍左右的性能。Stop Installing Tensorflow using pip for performance sake!、Optimizing TensorFlow for CPU

  4. github上也有一批issues发掘mkl优化在有些情况下反而影响性能。intel mkl optimized tensorflow performance degradation #23238

于是我进行了本地测试，情况如下：
（环境ubuntu 18.04，gcc 4.8.5，CPU为Intel i7-6700/Skylake架构）
（统一采用tensorflow最新1.13.1版本，个别预编译包为1.12.0版本）

  • pip安装无mkl的预编译包：
    • 简单矩阵运算：10s
    • word2vec：18s
    • seq2seq：34s
    • resnet：2.6/sec （越大越好）
  • pip安装有mkl的预编译包（intel-tensorflow）、conda安装有mkl的预编译包：
    （intel对此明确说是按照march=corei7-avx，即Sandy-Bridge架构）
    • 简单矩阵运算：5s
    • word2vec：40s
    • seq2seq：32s
    • resnet：2.6/sec
  • 自编译，按当前CPU优化（march=core-avx2）的无mkl的预编译包：
    • 简单矩阵运算：6s
    • word2vec：18s
    • seq2seq：26s
    • resnet：3.9/sec
  • 自编译，按当前CPU优化（march=core-avx2）的有mkl的预编译包：
    • 简单矩阵运算：5s
    • word2vec：40s
    • seq2seq：33s
    • resnet：4.0/sec

整体来说：

  • mkl对一些操作有优化，一些操作可能反而劣化。
  • 在有更好CPU情况下，用匹配的CPU指令集来编译，提升是较明显的。
