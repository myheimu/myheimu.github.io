Algorithm: #25. Reverse Nodes in k-Group

https://leetcode.com/problems/reverse-nodes-in-k-group/

问题：将单链表中每k个作为一组进行颠倒，只允许空间复杂度为O(1)

Example:
Given this linked list: 1->2->3->4->5
For k = 2, you should return: 2->1->4->3->5
For k = 3, you should return: 3->2->1->4->5

解答：在颠倒整个链表的算法基础上，采用几个指针，然后硬找规律。

Runtime: 12 ms, faster than 99.96% of C++ online submissions for Reverse Nodes in k-Group.
Memory Usage: 9.8 MB, less than 63.02% of C++ online submissions for Reverse Nodes in k-Group.

Review: 《How does a HashMap work in JAVA》

http://coding-geek.com/how-does-a-hashmap-work-in-java/

这篇文章详细讲解了Java中的HashMap。

HashMap是采用开链法处理冲突，因此Entry里面除了key和value之外，还有next指向链表的下一元素。另外为了防止重复计算hash值，还增加了hash字段。

对于hash值，并没有直接采用Object的hashCode，而是以此为基础，再进行hash，以避免原有hashCode的不均匀性。

HashMap默认设定容量因子loadFactor=0.75，如果达到，则会自动扩容，如果过小并不会自动缩容。

HashMap并不是线程安全的，原因是在自动扩容过程中会有很多中间状态，使得读写不一致。HashTable是线程安全版本，但锁粒度太大，所以在Java5之后新增了ConcurrentHashMap，作为线程安全的HashMap。

通常比较适合作为key的类型是String和Integer，因为他们不可变。如果采用可变的类型作为key，则很可能会丢失数据。

Java8中对HashMap做了较大优化，代码量上就从1k增加到了2k，其中结构上的最大变化是引进了红黑树，即：如果单个hash槽对应的key-value低于8个，则仍然采用链表形式存储，否则采用红黑树形式存储。

Tip: PCA两种算法的一致性

PCA 主成分分析，通常有SVD分解和协方差矩阵两种算法，这两种算法殊途同归，最终计算公式是相同的。

SVD分解解法

针对m个n维向量，采用行向量拼接，对每一行减去所有向量平均值（中心化预处理），构成矩阵 $A_{m×n}$

通过SVD分解：$A_{m×n}=U_{m×m}\Sigma_{m×n}V_{n×n}^T$

取前 r 大的奇异值：$A_{m×n}\approx U_{m×r}\Sigma_{r×r}V_{n×r}^T$

转换为 $A_{m×n}V_{n×r}\approx U_{m×r}\Sigma_{r×r}$，其中 $V_{n×r}$ 即为所求的可以进行特征压缩的转换矩阵

具体转变为求 $A^TA$ 的特征值，取前r大的特征值，他们所对应的特征向量即为所保留的主轴方向
