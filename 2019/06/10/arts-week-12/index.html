Algorithm: LeetCode #33. Search in Rotated Sorted Array

https://leetcode.com/problems/search-in-rotated-sorted-array/

问题：严格递增数组进行循环变换，然后寻找target值的位置

解答：二分法，时间复杂度 $O(\log n)$ ，关键是要注意二分的判断条件。

Runtime: 0 ms, faster than 100.00% of C++ online submissions for Search in Rotated Sorted Array.
Memory Usage: 8.8 MB, less than 47.65% of C++ online submissions for Search in Rotated Sorted Array.

Review: 《“Sequence to Sequence Learning with Neural Networks”: Paper Discussion》

https://hackernoon.com/sequence-to-sequence-learning-with-neural-networks-paper-discussion-6be16f19ecae

Seq2Seq模型是非常经典的端到端深度学习模型，最初用于展现神经网络机器翻译远超统计机器翻译的强大能力。

需要注意的是，在2014年这篇论文发表的时候，还没有Tensorflow、PyTorch这些工具，而DNN也才仅仅刚开始使用。

论文中的架构分为两个深度LSTM网络：

  • 一个作为Encoder，负责将输入序列转换到一个固定维度的向量上
  • 一个作为Decoder，负责将固定维度的向量转换到一个输出序列上

LSTM网络用于从输入序列预测输出序列的可能性：

两个LSTM网络的存在，是为了能够同步处理多种语言翻译对；论文中采用四层LSTM，充分展现了深度神经网络的能力；论文甚至尝试将输入序列进行逆序输入

训练细节：

  • 源语言采用16万的高频词，目标语言采用8万的高频词，其余未录入词采用特殊的UNK标记
  • 训练目标是最大化对数概率：1$/|\mathcal{S}| \sum_{(T, S) \in \mathcal{S}} \log p(T | S)$
  • 翻译结果的生成，采用最大可能的结果：$\hat{T}=\arg \max _{T} p(T | S)$
  • 采用获得输出特殊标记<EOS>作为结尾，进行贪心搜索
  • 所有LSTM参数采用-0.08到0.08之间的均匀分布初始化
  • 对梯度进行约束，避免梯度爆炸

最终在机器翻译的结果上，获得了长足的进步，首次体现了深度学习在机器翻译上的能力
